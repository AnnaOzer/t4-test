Установить OpenServer с поддержкой php7

Установить Composer
под Windows
- скачать файл
- сохранить себе
- путь к этому файлу внести в системную переменную path

Установим минимальное приложение на фреймворке T4
Проект Т4 это не только сам фреймворк,
но и стандартное приложение для него.
Сначалал задумывалось, что будет сам фреймворк и CMS на этом фреймворке.
CMS назвали просто - "стандартное приложение на фреймворке Т4"
Сейчас стандартное приложение в подвешенном состоянии, пилили и недопилили
поэтому сделали минимальное приложение,
т.е. некий минимум файлов, позволяющих созадть сайт на фреймворке Т4.

В консоли (OpenServer) дать команду

composer create-project pr-of-it/t4-app-mini --stability="dev" ./t4.local

пока нет стабильных версий поэтому пишем уровень стабильности = разработка
Композер спросит, не хотим ли удалить файлы гита, свн, ответить Да

Перезапустить OpenServer

Настройки ОпенСервера / Домены /
выбрать автопоиск доменов и самый максимально длинный вариант автопоиска корневой папки

Теперь набираем в адресной строке t4.local/
и видим
Hello, world!
It's T4 framework!

В папке public содержится точка входа в приложение, файл index.php
это единственный по сути публичный файл

в файле .htaccess пример конфигурации для веб-сервера apache
сам по себе готов к применению, лежит в правильном месте,
даёт правила перенаправления запрососв пользователя на index.php

файл nginx.conf - заготовка конфигурации для веб-сервера nginx,
содержимое его еще надо скопировать и вставить в конф файл nginx

веб-сервер должен смотреть на папку public
OpenServer делает это автоматически
если вручную настраивать - не писать чтобы веб-сервер смотрел на всю папку нашего проекта
