есть спец класса для конфигов
основан ка классе Std
в нём работает fromArray

в конструктор можете передаваться массив или путь к файлу
реализован Active Record,
позволяющий менять содержание файла конфига

Active Record в t4 такой интерфейс он задаёт
2 метода
save(сохраняет по старому месту - файл или БД) и delete (будет удален файл целиком)
позволяет в простейших случаях обойтись без БД
мы можем задать некий конфиг файл как
место хранения каких-то данных
в админке эти данные показывать в форме
из формы их встречать
записывать снова в конфиг
говорить save
и они снова автоматически запишутся в этот файл

где-то в глубине используется функция var_export
когда сохраняем в файл
