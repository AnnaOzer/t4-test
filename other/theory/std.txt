------------------------------------------------
Std - сокращение от Standard

Основной класс фреймворка, основа многих других классов.
Его основные характеристики:
- динамические свойства, создаются на лету
(у объекта этого класса могут быть совершенно любые свойства)
- создание цепочек
- возможность написать геттеры и сеттеры
- sanitize данных при set
(суть санитации
можете приводить значения к нужному вам виду)
- validate данных при set
(суть валидации
мы можете запретить присваивание каких-либо значений
свойствам своих объектов
т.е. разрешать или запрещать)
- полиморфизм объект-массив
(часто требуется в Твиге, шаблонизваторе)
- реализация интерфейса IArrayable
- реализация Countable
- итерация по динамическим свойствам

(isset, unset - всё это в нём есть)

В фреймворке имеется специальный трейт
TStdGetSet, позволяющий подмешать поведение стандартного класса к любому вашему классу!

в Std качественно реализован array object
концепция, когда объект и массив изоморфны

-----
реализован интерфейс IArrayable
есть метод fromArray, создающий объект из массива
метод toArray превращает объект в обычный простой массив

-----
$obj->foo->bar->baz = 42;
можем задать цепочку свойств
каждое промежуточное свойство тоже объект класса того же, что и сам объект
(в примере - класса App/Components/Test)
примеры возможных цепочек
$this->data->webinars->new
$this->data->webinars->finished

----
если в проекте своя система наследования
можн классу скажем Test подмешать трейт
вот так
use TStdGetSet;
и ваш класс станет аналогом класса Std

пример на валидацию и санитацию
это валидный телефонный номер:
'+7 910 660 00 00'
валидация это вопрос валидно ли значение
и если ваш валидатор ответит да, то значение будет записано

а санитация это приведение к стандартному виду
такому
'79106600000'
а записать в базу мы должны вот это значение
