после установки

константы
содержат пути в файловой системе

ROOT_PATH
ROOT_PATH_PUBLIC
ROOT_PATH_PROTECTED

Не использовать $_SERVER['DOCUMENT_ROOT'] тк может быть перенаправление и само приложение вообще не должно там находиться

-------------------------

index.php менять не надо вообще никогда

boot.php не рекомендуется менять

autoload.php
показываем, что все классы будут находиться в пространстве имён App

в index.php
строчки
T4\Mvc\Application
    ::instance()
означают
::instance() признак синглтона, статический метод для получения экземпляра объекта
класса T4\Mvc\Application
т.е. есть объект приложение и с помощью этого инстанс он у нас получается, создаётся

дальше мы задаём ему конфиг
->setConfig(
        new \T4\Core\Config(ROOT_PATH_PROTECTED . '/config.php')
    )
создаётся специальный оюъект, которому указывается, где конфиг находится

в данный момент конфиг нашего приложения пуст и представляет собой return []

    ->run();
метод, который начинает выполнять наше приложение

еще раз
- получаем экземпляр приложения
- задаём ему конфиг
- запускаем
