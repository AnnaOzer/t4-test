Миграции -  способ записать в виде кода те изменения, которые производите с БД

чтобы хранить их там, где вы храните свой основной код
и коллегам было видно

1)
в терминале
php protected/t4.php /migrate/create --name=createPersons
миграция для заведения в базе таблицы persons
создает файл с классом миграции с заданным именем


в папке protected появляется папка Migrations
это класс с 2 методами up() применяет и down() отменяет нужные изменения в базе

2)
заполнить кодом методы up() и down() миграции, которые, соответственно, применят и отменят изменения

см. в коде vendor/pr-of-it/t4/framework/Orm/Migration.php много полезных методов
вроде createTable(), addColumn()

3)
php protected\t4.php /migrate
применит все непримененные ранее миграции

вы это закоммитите в гит, вася и петя скажут у себя также т4 мигрейт
при выкладке сайта в бой также надо запустить команду т4 мигрейт и больше ничего делать не надо

4)
php protected\t4.php /migrate/down
отменит одну последнюю миграцию

Миграции позволяют:

контролировать кто, когда и зачем изменил базу данных (например - с помощью git)
изменять структуру и ключевые данные
автоматически применять изменения при деплое проекта
развертывание на целевом сервере - обновить файлы, запустить миграции, всё

NB миграции - это часть консольного приложения на T4

**********
типы полей в разных БД разные
т4 поддерживает универсальные типы данных
string int float date time
которые для каждой бд превращаются в её родные
